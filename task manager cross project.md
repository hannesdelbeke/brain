my [[task management]] systems need to work cross project
- one project might store tasks in its repo
- another stores tasks in [[github issue|github issues]]
- another uses my [[work]]'s [[task manager]] like [[JIRA]]

AI can hook this all up with [[MCP server|MCP]] or connectors, but you still need to establish some kind of personal overview.

In that overview, i want to see an overview of tasks i did:
- 5-6 pm task A for work - task lives in work JIRA
- 6-6.30 pm cook dinner - task lives in todoist i share with partner
- 6.30 -7 personal code project - task lives in repo

to [[visualize time spent]], it would be great to see:
- time blocks in a weekly/daily [[calendar]] -> what did i do today?
- duration blocks over days in a month -> how long it took to complete a project, a task, an epic?
- see how much of your time usually goes to chores, to work, to a certain project.

**each system needs a priority system.**
- at work, i need to do these 4 tasks this week
- at home i need to do these 3 chores this week and this 1 chore today
- at personal project, this is the order of tasks. 

**each system needs a ranking compared to other systems**
- the whole personal project is not important compared to work or personal life tasks such as dentist appointment.
- work is more important than chores, but less important than medical appointments.
